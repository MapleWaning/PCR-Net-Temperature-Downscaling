# Data and Release Artifacts

## Data Levels

The repository uses three data levels.

| Level | Location | Purpose | Reproducibility role |
|---|---|---|---|
| Minimal fixture | Generated by Demo 1 and partially tracked as `smoke_case` | Interface and smoke testing | Software smoke test only |
| Release mini-case | `assets/demo_data/mini_case/` | Public workflow validation | Public demo reproduction |
| Full research dataset | User-prepared `data/processed/` layout | Manuscript experiments | Manuscript numerical reproduction |

The complete research dataset contains 118 stations over 2008-2024. It is not fully redistributed because source products remain subject to third-party access procedures, licences, terms of use, and storage constraints.

## Mini-Case Dataset

The Release `mini_case` is a reduced real subset derived from the research data. Verified metadata:

- Stations: 12.
- Years: 2011, 2014, 2016, 2018, 2020, 2023.
- Dates per year: 01-15, 04-15, 07-20, 11-10.
- Sample universe: 288 samples.
- Patch size: 128 x 128.
- Temperature channels: mean, maximum, minimum.
- Temporal split: train years 2014, 2018, 2020; validation year 2016; test years 2011 and 2023.
- Spatial split counts: 6 train stations, 2 validation stations, 4 test stations.

The HDF5 files keep full year-length day axes for compatibility. Only selected date/station chunks are intended for demo use.

## Download Commands

```bash
python scripts/download_release_artifact.py mini_case
python scripts/download_release_artifact.py pcr_time
python scripts/download_release_artifact.py pcr_spatial
```

Download all Release artifacts:

```bash
python scripts/download_release_artifact.py all
```

The script verifies file size and SHA256 before installing artifacts. Interrupted downloads may leave `.part` files; verified partial files can still be used by the installer.

## Pretrained Checkpoints

`pcr-time.pth` is used for temporal-generalization runs such as `--version time` or `--version temporal`.

`pcr-spatial.pth` is used for spatial leave-out runs such as `--version spatial`.

Demo 3 requests the matching checkpoint automatically. If a checkpoint is missing and no explicit `--model-path` is supplied, the demo invokes the Release downloader.
